# <a name="remove-a-legacy-authorized-host-in-lync-server-2013-optional"></a>Entfernen eines autorisierten Legacyhosts in lync Server 2013 (optional)

<div data-xmlns="http://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="http://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

<div data-asp="https://msdn2.microsoft.com/asp">



</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Letztes Änderungsstand des Themas:** 2012-09-05_

Wenn Sie die Remoteanrufsteuerung zu einer lync Server-Bereitstellung migrieren, müssen Sie Legacy-autorisierte Hosteinträge (als *Vertrauenswürdige Anwendungs Einträge* in lync Server bezeichnet) für alle SIP/CSTA-Gateways in Ihrer Legacy Bereitstellung entfernen. Zum Entfernen von Einträgen für autorisierte Hosts aus diesen Bereitstellungen müssen Sie die Verwaltungstools verwenden.
