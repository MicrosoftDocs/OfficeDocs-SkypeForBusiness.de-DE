> [!IMPORTANT]
> <span data-ttu-id="67d0a-107">Damit die Reporting Services-Berichte erfolgreich bereitgestellt werden können und nach Abschluss der Bereitstellung auf die Berichte zugegriffen werden kann, muss TCP/IP-Port 80 (und optional TCP-Port 443 für SSL vorhanden sein, wenn Sie dem Reporting Services ein Zertifikat zuweisen), das in der Windows-Firewall mit erweiterter Sicherheit auf dem SQL Server geöffnet ist.</span><span class="sxs-lookup"><span data-stu-id="67d0a-107">For the Reporting Services reports to deploy successfully, and to access the reports after deployment is complete, you must have TCP/IP port 80 (and optionally, TCP port 443 for SSL, if you assign a certificate to the Reporting Services) open in the Windows Firewall with Advanced Security on the SQL Server.</span></span> <span data-ttu-id="67d0a-108">Ausführliche Informationen finden Sie unter [Konfigurieren der Windows-Firewall, damit SQL Server Access](https://go.microsoft.com/fwlink/p/?linkId=218031) für Microsoft SQL Server 2008 R2 zugelassen wird.</span><span class="sxs-lookup"><span data-stu-id="67d0a-108">For details, see [Configure the Windows Firewall to Allow SQL Server Access](https://go.microsoft.com/fwlink/p/?linkId=218031) for Microsoft SQL Server 2008 R2.</span></span>
  
<span data-ttu-id="67d0a-109">Nachdem Sie die Zusammenfassung überprüft haben, klicken Sie auf **Fertig stellen** , um die Installation der Berichte für SQL Server Reporting Services abzuschließen.</span><span class="sxs-lookup"><span data-stu-id="67d0a-109">After reviewing the summary, click **Finish** to complete the installation of the reports to the SQL Server Reporting Services.</span></span>
